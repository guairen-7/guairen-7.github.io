【实验六】【为用户设置权限】

2022-05-17

一、在“安全性”里设置用户权限

①打开“SQL Server Management Studio”的“对象资源管理器”窗口，即打开SSMS时自动连接的窗口。

②选择“本地服务器”— “安全性”— “登录名”选项，
鼠标右键单击登录名，在弹出的快捷键菜单里选择“新建登录名”选项。

③在弹出的登录名对话框的常规页面，建立登录名“abc”。

④在选中数据库前的复选框之后，“数据库角色成员身份”区域里的“public”复选框里会被自动选择上。在每个数据库中，所有用户都会默认是public角色的成员，并且不能被删除。

身份验证选择“SQL Server 身份验证”并设置密码：

二、在数据库里设置用户权限

sa用户与超级用户：
如果只启用了SQL身份验证模式，超级用户就是sa，密码是你安装时设的。
如果只启用了Windows集成验证模式，服务器上Administrators组的所有用户都是超级用户，即默认登录的用户即可。
如果启用了混合模式就包括了上面两种用户。

①启动“SQL Server Management Stuido”，以sa用户或超级用户身份连上数据库实例。在“对象资源管理器”里选择“数据库实例名”— “XSCJ”— “安全性”— “用户”。鼠标右键单击用户，在弹出的快捷键菜单里选择“新建用户”选项。

②在弹出的弹出“数据库用户”对话框的常规页面，建立用户“userl”。

这里默认选择“带登录名的SQL用户”，输入用户名与登录名后，在新建用户时会出现报错15007，做以下修改：选择“不带登录名的SQL用户”

③在该对话框里选择“安全对象”标签，弹出对话框，单击“搜索”按钮，弹出 “添加对象”对话框，在该对话框里可以选择希望查看的对象类型的选择对话框。

④上一步点击“确定”后，弹出 “选择对象”对话框，在该对话框里单击“对象类型”按钮。

⑤单击“对象类型”按钮后，弹出 “选择对象类型”对话框，在该对话框里可出现在数据库表里的可用对象类型，在本例中选择“表”复选框，然后单击“确定”按钮。

⑥单击“浏览”按钮。

⑦在弹出 “选择对象”对话框，单击列表中的XSQK表，两次“确定”，通过“数据库用户”对话框下半部分的权限列表进行授权，为数据库用户userl 设置XSQK表上的Alter、Delete、Insert、Select和Update权限，单击“确定”按钮完成。
