【与不确定性和解】

开始将这些思考记录到博客，目的是：

  • 锻炼文笔（新知识输入少、看书少、陷入信息茧房）
  • 逻辑能力（因为脑子里无用的信息很多，最近发现答非所问现象严重）
  • 语言表达能力（表达能力逐渐变差，常用词语在短期内形成了习惯，缺乏输入）
  • 改变就是好事

记录背景

  1. GET-4没过
  2. 与学姐的谈话–与不确定性和解
  3. 朝着大佬看齐–GeekPlus、衣戈猜想
  4. S12 QuarterFinals DRX
  5. 刷了《小明与王猫》
  6. 时间静止3week

前言：外在表现背后存在原因，挖掘现象背后的本质，想要改变不难，方法总比困难多

生活上

  1. 拖延症
    炎症：觉得某些东西有用但没立即学

  2. 缺乏自信、恶性循环
    炎症：不表现情感、自我心理限制直接表现为行为自我限制

  3. 盲目的累
    炎症：一天过去收获与身心上的疲劳不成正比

  4. 时间焦虑

    炎症：刷视频焦虑，漫无目的地刷、无休无止地刷。对未来工作的焦虑，对身边没有参考案例的未知感强烈

  5. 信息茧房
    炎症：进入脑子的知识越来越少，对外输出少之又少

心理上

  1. 拖延
    发炎处：没有意识到事务的价值
    良药：事务分轻重缓急、分解任务的同时对不确定性加以认知

  2. 自我否定
    发炎处：没有理解人的多元性
    良药：理解成长性、多元性，肯定（v.）收获并加以正循环，心理是会成长的

  3. 盲目的累
    发炎处：小事烦扰、没有学到本质、无故的内耗
    自己开药：定时处理掉无用的信息、将重心放到想要利用它完成的具体事件中（即有目标）
    附：钱颖一：我从清华学生中观察到的七个普遍现象/Datawhale

  4. 浮躁
    发炎处：
    良药：未知、不确定性是通性、预见性仅供参考、活在当下

  5. 信息茧房
    良药：RSS（订阅万物、脱离推荐算法）、读书。

学习上

1.GET-4

  • 外在原因：暂无（惭愧地说这是大学里面第一次考，但“知己知彼”：对时间安排、考试内容、考试流程都比较了解，没有出现特殊情况，因此想不出外在原因）
  • 内在原因：没复习、裸考
  • 从各个角度分析&总结&改变：

  1. 对英语的认知：
    我在小初高阶段都是靠一种语感走过来的，比如一个陌生句式我不会刻意地去分析（例如一个见得最多的：Only in this way will you make it.从语言的角度来看，英语地教育经常使用总结来归并句式，当然这里是我的造诣不深），心里总反问：“这个句式不就该是这样的么？为什么要去分析它？”
    对于语言的学习我很认同Ta们的见解：
    丁老师：语言的学习跟日常联系紧密
    小妹：语言不应归并为一门科目
    通老师：语言是一种工具，而考试的性质是考试

  2. 考前状态：
    时间：2022.9.1-9.2返校，个人延迟至9.10返校，学校于9.17举行GET考试。返校后的一周时间也是没有看真题，保护一下心态，可以说是时隔两年的裸考了。
    心态：临时抱佛脚肯定没用，上学期的计算机英语我也是这样想的，因此平时积累与复习就好了。
    “知己知彼”：对考试流程、时间安排、题型时间分配、设备调试等都很了解，因此没有外在失利因素。

  3. 考后感觉：
    题目难度：与高考难度相差不大，但因为太久没做英语题，手生疏了
    时间分配：较为合理，高考时期我对英语考试中的时间分配也特别清晰
    是否能过：听力与阅读等选择题存在不确定性，过线只是一种可能

题型分析见：【GET-4】

  1. 如何对症下药：
    磨耳朵：这点对于习惯高中听力练习模式的我并不排斥，因此备考过程要有成绩追求地练习
    适当提高阅读技巧：针对不同题型提高答题速度
    写作&翻译：练习字体及中高级词汇

  2. 时间紧迫：
    理想状态是2022年上半年过GET-4，下半年过GET-6。2023年上半年准备备考，时间上不多。因此要抓住2022下半年的GET-4。

2.日常作业

前言：一直以来没有总结过对待科目的感性认知，曾经都是不温不火的态度。这学期对待一些科目的反差还是比较大的，因此理性分析+找到对策：

科目 看法 存在的问题 优势 如何解决
ML 本学期最爱，因为本身就充满多样性，自由，学习成果很容易从输出框获得反馈 暂时没有自己的一套解决方法&喜欢C+V 喜欢动手、没有焦虑感 在尝试别人的代码后要善于总结
Java 很喜欢通老师的讲课风格，他会将自己的见解、理解传授给我们，以及如何去从课本外提升代码素养 缺乏课后的巩固、主动动手少 逐渐对代码产生兴趣（因为理解更多以前看不懂的部分） 肢解每一个“大任务”，一点点来做、不要焦虑、注重项目的逻辑、不要“自洽”编程的相异性
NoSQL 课时短、喜欢实验动手，完成一些新内容的实践很容易获得成就感以及对相关知识的渴望 SQL语言基础不牢固、“debug”能力有待提高 喜欢动手做、喜欢新鲜的技术 见到报错不要慌，找到解决方法后去尝试，注意不要陷入多个解决方法同时进行的局面
R 前两年很多课程涉及、很容易从完成代码角度获得反馈 没有参考、被喂习惯了 了解R语言的一些基础知识（不包括内容上） 将基础知识进行总结为文章，解决问题时静下来，即使是搜索也要静下来，无论通过什么方法学到心里即可
Python 有过一些实践经验、课本内容太简单，就像几篇博客就能总结下来 有些基础课后题不能够独立完成、有些常见代码不懂含义 写过一些代码（但不多） 就着兴趣来，整理课后题与常见代码文章
云计算 理论内容更像是冗余，理论框架的建设必不可少，但有些长篇大论的理论有时不如一张抽象的图片 理论枯燥 喜欢开放的演讲氛围、喜欢阿里云起实验室平台的模式 通过文章总结云计算的理论框架、实验不在话下，但要注意主观能动性
计组 在学之前兴趣最大的一门课程，迫切想要知道计算机的运行原理 教材与市面主流不一致、两个专业底子不同、老师会忽视掉很多这方面的问题、讲的内容我能理解到她会，但真的听不懂 探索欲望还是依存的 哈工大的课程尽快过一遍、有问题直接问同学
算法 理想中应该是最热爱的一门课程，但恰恰相反 上课的人数太多、PPT型老师、很久没有代码上的成就感、底子差、debug能力太差 看到代码能够运行起来还是后成就感的 注意总结每块算法的本质与理解、勤加练习

  2. 分解任务，而且要明确如何去做、通过什么去做
